# Axiom-Physics

Axiom-Physics is a minimalist 2D physics module for C++ with a clear separation between `Body`, colliders, and a registration-based `PhysicsWorld`.

## Building

Project files are generated by [Premake5](https://premake.github.io/). The bundled binary lives at `vendor/bin/premake5.exe` so no extra installation is required.

### Windows (Visual Studio 2022)
```
Setup.bat
```
This invokes Premake to produce `Axiom-Physics.sln`, the static-library project, and the `Tests` console project. Open the solution and build, or invoke MSBuild directly:
```
msbuild Axiom-Physics.sln /p:Configuration=Debug   /p:Platform=x64
msbuild Axiom-Physics.sln /p:Configuration=Release /p:Platform=x64
msbuild Axiom-Physics.sln /p:Configuration=Dist    /p:Platform=x64
```

### Other Premake actions
`Setup.bat` forwards its first argument to Premake, so any supported action works:
```
Setup.bat vs2022
Setup.bat gmake2
Setup.bat xcode4
```

The generated solution, project files, and `bin/` / `bin-int/` output directories are gitignored — they are pure build artifacts.

### Running tests
The default startup project is `Tests`. After a build, run:
```
bin\Debug-windows-x86_64\Tests\Tests.exe
```
The runner executes every registered case and reports a summary; exit code is non-zero on any failure.

## Build target
Axiom-Physics is configured as a **static library** in all configurations (`Debug`, `Release`, `Dist`) on the `x64` platform.

- Project output type: `StaticLib`
- API macro: `AXIOM_PHYS_API` is empty for static builds. Define `AXIOM_PHYS_BUILD_DLL` (when building) or `AXIOM_PHYS_IMPORT_DLL` (when consuming) to switch to a DLL build.
- Per-config defines: `AXIOM_PHYS_DEBUG`, `AXIOM_PHYS_RELEASE`, `AXIOM_PHYS_DIST`.
- Warnings: `Extra` (W4 on MSVC, `-Wall -Wextra` elsewhere).

## Architecture
- **Body**: A single `Body` class with `BodyType` (`Static`, `Dynamic`, `Kinematic`). Carries position, velocity, mass, and restitution.
- **Colliders**: `BoxCollider`, `CircleCollider`, and `PolygonCollider` derive from `Collider`. A body owns at most one collider; the bidirectional pairing is established by `PhysicsWorld::AttachCollider`.
- **World**: `PhysicsWorld` stores global settings, registration logic, integration, broadphase (uniform spatial hash), narrowphase, and resolution.
- **Queries**: `Physics2D` provides overlap and point queries with and without an active world context. All queries return values (`std::optional<Contact>`), never references to internal storage.

## Behavior
- Only bodies and colliders registered in `PhysicsWorld` are simulated.
- Gravity affects only bodies of type `BodyType::Dynamic` (when gravity is enabled for that body).
- World bounds can be enabled globally; per-body boundary checks can be toggled.
- Collision normal points from `bodyA` toward `bodyB`; resolution moves A by `-normal * penetration` and applies a velocity impulse using `min(restitutionA, restitutionB)`.
- Narrowphase: precise circle-vs-circle (distance test) and circle-vs-box (closest-point test). Box-vs-box and any pair involving a polygon fall back to AABB overlap.
- `SetBodyType` applies sensible defaults:
  - `Static`: gravity off, boundary checks off, velocity reset to zero
  - `Kinematic`: gravity off, boundary checks on
  - `Dynamic`: gravity on, boundary checks on

## Example
```cpp
#include "PhysicsWorld.hpp"
#include "Body.hpp"
#include "BoxCollider.hpp"

int main()
{
    AxiomPhys::PhysicsWorld world;

    AxiomPhys::Body player(AxiomPhys::BodyType::Dynamic);
    player.SetPosition({ 0.0f, 2.0f });

    AxiomPhys::BoxCollider playerCollider({ 0.5f, 1.0f });

    world.RegisterBody(player);
    world.RegisterCollider(playerCollider);
    world.AttachCollider(player, playerCollider);

    world.Step(1.0f / 60.0f);
    return 0;
}
```
